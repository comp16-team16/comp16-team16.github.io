2. Implementing A Lightweight Reactive Framework

Introduction

In this post, we'll be implementing a lightweight framework that'll allow us to implement our app's UI reactively. In web development, reactivity means keeping the view and model (state) automatically in sync. It's often used synonymously with declarative UI design, where sequential instructions are replaced with more descriptive functional code. When used together, these paradigms allow us to drastically increase code clarity and maintainability. However, unfortunately JavaScript supports neither of them natively. As such, most modern web apps are developed with rather large frameworks such as React or Angular which add layer of abstraction to support these paradigms. In our case, we've already outlined why we won't be using such frameworks. This of course leaves us with the problem of achieving reactivity natively.

Variable Fibers

Our approach will revolve around a slightly more advanced kind of variable, called a variable fiber. Fibers are like regular variables, however they're values can be observed. Whenever a fiber's value changes, all observers are notified.

Variable fiber implementation:

class VariableFiber {

	constructor(value) {
		this.value = value
		this.observers = []
	}

	change(value) {
		if (value !== undefined) this.notify(
			this.value = value
		)

		return this.value
	}

	notify(value) {
		this.observers.forEach(
			observer => observer(value)
		)
	}

	on(observer) {
		observer(this.value)
		this.observers.push(observer)
	}
}

And that's all we need! Before continuing, let's reimplement this as a functional interface to make the API more concise:


function VariableFiber(value) {
	const observers = []

	function interface(change) {
		if (change !== undefined) notify(value = change)

		return value
	}

	function notify() {
		observers.forEach(
			observer => observer(value)
		)
	}

	interface.on = function(observer) {
		observer(value)
		observers.push(observer)
	}

	return interface
}

Let's test it out:

const name = VariableFiber('Alice')

name.on(name => console.log(`name is currently ${name}`))

// console outputs "name is currently Alice"

name('Sam')

// console outputs "name is currenlty Bob"

Perfect! These fibers will act as a building block for adding reactivity into our UI.

Declarative components

In addition to reactivity, we would like our UI components to be highly declarative. For example, consider the following widget:

One way to create this widget would be to procedurally build each DOM node (the buttons, inputs etc), then apply styles, and then add any event listeners needed. All in all, this process is time consuming when programming and leads to long, often hard to read sequential code. What we would really like is to be able to just describe the layout like we do in HTML; any styles can be described separately with CSS and any event listeners can be declared as node attributes. This leads to far more concise, readable and maintainable code. However, again, JavaScript provides no way to build DOM trees like this. Our solution is to use nested function calls to construct the tree and JavaScript objects to supply node attributes. This method is largely inspired by Flutter.

To implement this, we just need a small function for removing boilerplate:

export default function(literal) {

	return props => {

		const style = document.createElement('style')
		const host = document.createElement('div')
		const root = host.attachShadow({ mode: 'open' })

		Object.assign(host, props, literal)

		style.append(...host.styles())
		root.append(style, host.template())

		return host
	}
}
